Boombox documentation

Firmware project for a LILYGO T-Display (classic ESP32), destined to become the “Digital Tape” module of a Panasonic RX-5235 boombox. These pages are self-contained HTML — no network, scripts, or frameworks needed to read them.

  • RX-5235 “Digital Tape” build & integration plan

    The living execution plan: current-state snapshot, milestone dashboard M0–M12, all build phases with pass/fail and stop conditions, wiring tables, test matrices, inspection worksheets, audio/power design, firmware breakdown, and troubleshooting — sourced and classified.

  • M1 — USB-only Bluetooth bring-up plan

    The detailed, bench-usable expansion of milestone M1 / Phase 3 of the build plan: flash the pinned A2DP-sink example in its audio-discarding “Idle” mode and prove pairing, connection, and stream state over USB alone — no DAC, no I2S pins, no audio. Includes the safety preflight, source-grounded config delta, command sequence, evidence checklist, and troubleshooting.

  • Toolchain & project setup research

    Why this project uses native ESP-IDF pinned at v6.0.2, with cited comparisons against PlatformIO and Arduino, the macOS setup path, and every deferred decision (partitions/OTA, secrets, testing, CI).

  • Hardware notes & identification status

    What is verified about the attached device, the board photos and the pin review that gate firmware work, the identification checklist, and the safe-workflow rules for flashing.

  • ESP32-D0WDQ6-V3 chip architecture

    Die-level facts from the official datasheet and TRM: CPUs, memory map, boot path, ULP/RTC domain, radios, DMA and I2S — and the boundary where verified SoC facts end and unknown carrier wiring begins.
